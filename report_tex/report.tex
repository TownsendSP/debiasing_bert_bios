	\section{Introduction}
	This project investigates the lifecycle of gender bias in NLP models. By training a model from scratch and then fine-tuning it under different constraints, we identify how linguistic representations can both inherit and mitigate societal disparities.

	\section{Phase 3 \& 4: Comparison of Biased and Debiased Models}
	The ``Biased'' model was trained with stereotypical oversampling, while the ``Debiased'' model used counterfactual augmentation and balancing.
	
	\subsection{Fine-tuning Architecture and Process}
	Starting from the pre-trained Raw BERT checkpoint, we add a lightweight classification head and fine-tune the entire network end-to-end for profession classification across 28 categories.
	
	\paragraph{Classification head.}
	The \texttt{[CLS]} token's 768-dimensional hidden state is passed through a dropout layer ($p = 0.1$) and then a single linear projection to 28 logits, one per profession.  During inference the logit vector is passed through a softmax and the \texttt{argmax} selects the predicted profession.
	
	\begin{figure}[ht]
		\centering
		\begin{tikzpicture}[node distance=0.7cm, every node/.style={font=\small}]
			
			% Input tokens
			\node[datablock, text width=5.8cm] (input)
			{\texttt{[CLS]\ \ She\ \ graduated\ \ from\ \ Harvard\ \ Law\ \ \ldots}};
			
			% BERT encoder
			\node[wideblock, below=of input, fill=blue!18, text width=5.8cm] (encoder)
			{BERT Encoder\\(14 layers, 768-dim hidden, 12 heads)};
			
			% Hidden states
			\node[datablock, below=of encoder, text width=5.8cm] (hidden)
			{Hidden states: $\mathbf{h}_{\texttt{CLS}},\,\mathbf{h}_1,\,\mathbf{h}_2,\,\ldots$};
			
			% CLS extraction
			\node[block, below=of hidden, fill=yellow!20] (cls)
			{Extract $\mathbf{h}_{\texttt{CLS}} \in \mathbb{R}^{768}$};
			
			% Dropout
			\node[block, below=of cls, fill=gray!15] (drop)
			{Dropout ($p=0.1$)};
			
			% Linear
			\node[block, below=of drop, fill=orange!20] (linear)
			{Linear $768 \!\to\! 28$\\(classification head)};
			
			% Softmax
			\node[block, below=of linear, fill=green!15] (softmax)
			{Softmax $\to$ argmax};
			
			% Output
			\node[outblock, below=of softmax] (output)

			\draw[arrow] (cls)     -- (drop);
			\draw[arrow] (drop)    -- (linear);
			\draw[arrow] (linear)  -- (softmax);
			\draw[arrow] (softmax) -- (output);
			
		\end{tikzpicture}

	\paragraph{Training settings.}
	Both the biased and debiased models share the same hyper-parameters: AdamW optimiser, learning rate $2 \times 10^{-5}$ with linear warm-up (6\% of steps), weight decay $0.01$, mixed-precision (\texttt{fp16}), gradient clipping at 1.0, and 3 fine-tuning epochs.  The sole difference is the composition of their training sets.
	
	\subsection{Debiasing Process and Counterfactual Augmentation}
	\label{sec:debiasing}
	
	The debiased training pipeline applies two complementary interventions before any gradient step is taken.
	
	\subsubsection*{Step 1 — Gender Balancing}
	For every profession class, we independently sub-sample the male and female subsets to their common minimum count, producing a dataset that is perfectly balanced across gender within each profession.
	
	\subsubsection*{Step 2 — Counterfactual Data Augmentation (CDA)}
	Each biography in the balanced set is duplicated with its gendered cues swapped: pronouns (\textit{he} $\leftrightarrow$ \textit{she}, \textit{him} $\leftrightarrow$ \textit{her}, etc.), titles (\textit{Mr.} $\leftrightarrow$ \textit{Ms.}), and a curated list of gendered kinship/role words (\textit{father} $\leftrightarrow$ \textit{mother}, \textit{brother} $\leftrightarrow$ \textit{sister}, \ldots).  The profession label is preserved; only the gender label is flipped.  This teaches BERT that the same biography could belong to a person of either gender, preventing it from using gender as a predictive shortcut.
	
	Figure~\ref{fig:cda} illustrates this augmentation step.
	
	\begin{figure}[ht]
		\centering
		\begin{tikzpicture}[node distance=1.0cm and 1.6cm]
			
			% Original bio
			\node[datablock, text width=5.2cm] (orig)

				\small``\textit{\textbf{He} received \textbf{his} M.D.\ from Johns Hopkins.

	\subsection{Performance Breakdown}
	A critical component of our evaluation involved testing the model on ``Counter-Stereotypical'' samples. For example: female surgeons or male nurses.
	
	Our analysis shown in Figure 27 reveals a stark contrast in performance. The biased model maintained high accuracy in stereotypical samples but saw a sharp decline when encountering counter-stereotypical data. This confirms that the model was using gendered linguistic cues as ``shortcuts'' for professional classification.
	
	By implementing counterfactual data augmentation, The debiased model achieved a much more uniform accuracy distribution. As shown in Figure 20, the per-profession gender accuracy gap for roles like ``Surgeon'' and ``Software Engineer'' narrowed significantly. While this led to a negligible decrease in overall accuracy (from 69.5\% to 69.0\%), the trade-off is justified by the drastic reduction in the Average Gender Gap, which fell from 26.1\% to 5.1\%, as shown in Figure 18.

	\section{Conclusion}
	By incorporating all 30 performance and bias metrics, we can see a clear trend: counterfactual data augmentation significantly improves model fairness without a substantial loss in predictive power. It demonstrated that LLM-based pipelines inherit systemic biases from datasets like Bias-in-Bios, where professions such as ``Dietitian'' and ``Nurse'' are heavily female-skewed, while ``Surgeon'' and ``Software Engineer'' remain male-dominated.
	
	Our results prove that Counterfactual Data Augmentation is a highly effective mitigation strategy. By balancing the training distribution, we successfully decoupled gender indicators from professional labels.
